# Agent-native

`idlefy-universal` carries first-class metadata for AI agents:
`x-agent-*` keywords on every schema node, plus an `agent-index.json`
file shipped inside the chart. This page explains the design choices.
For the consumer contract see
[Reference → agent-metadata](../reference/agent-metadata.md).

## Delivery-agnostic

The metadata is plain JSON Schema vendor keywords. Any tool that reads
the schema can consume it: a Claude Code skill, a docs generator, a
RAG pipeline, an MCP server, an IDE extension. The chart does not bind
to a particular agent framework. Adding support is a parsing job, not
an integration.

## Why JSON Schema, not a sidecar file

Two reasons:

1. **Cohesion.** The metadata lives next to the type it describes.
   Renaming a field, deleting it, or adding a constraint moves the
   agent prose with it. No second file to drift out of sync.
2. **Editor support.** JSON Schema language servers surface vendor
   keywords automatically — hover an `x-agent-when-to-use` value in
   VS Code's YAML extension and it renders. No editor configuration
   needed.

## Why also `agent-index.json`

An agent that wants the full surface area at once shouldn't have to
parse the schema's `$ref` graph and `if/then/else` chains. The index
flattens every field into one record:

```
{ "pointer": "/deployments/{name}/autoCreateRbac",
  "whenToUse": "…",
  "relatedFields": ["…"],
  "commonMistakes": ["…"],
  "exampleUseCase": "…" }
```

The schema is for editor-time interactive use; the index is for batch
agent ingestion. Both ship together. Both are generated by
`schema/build.py` from the same `schema/docs/*.yaml` source.

## What this is not

- **Not an agent SDK.** There is no client library, no transport, no
  authentication layer. The chart ships data; consumers do whatever
  they like with it.
- **Not a separate audience tab in the docs site.** Engineers and
  agents read the same Reference. There is no "for agents" sub-site;
  the agent material is woven into the contract.
- **Not framework-specific.** No coupling to Anthropic Skills, MCP,
  OpenAI tools, or any other delivery mechanism.

## What this enables

A skill that picks a workload kind, suggests `autoCreate*` flags, and
flags common mistakes — written against `agent-index.json` — works
against any chart version 1.0+ without bespoke logic per release.

## Trust chain {#trust-chain}

`idlefy-universal` publishes signed artifacts — every OCI chart release
is accompanied by a Sigstore cosign signature, a SPDX SBOM, and a
SLSA L3 build provenance attestation. An AI agent must verify all
three before `helm install`; this is the difference between "I trust
the registry I pulled from" and "I trust this specific chart, signed
by this specific workflow, with this specific dependency tree."

See [How-To → Verify the chart's supply chain](../how-to/verify-supply-chain.md#three-command-verification)
for the three-command verification recipe and the autonomous-agent
gate script.
